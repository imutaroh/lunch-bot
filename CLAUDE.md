# CLAUDE.md — lunch-bot

このリポジトリは Slack ランチシャッフル bot の **実運用コード + その読解学習** を集約する。

## このリポジトリの目的

Phase 1〜2 で AI 主導 (vibe coding) で作った lunch-bot を、**新卒・未経験エンジニアの読解教材として骨の髄まで解析する**。最終ゴールは:

- 全コードを自分の言葉で説明できる
- ETL構造としての lunch-bot を図示できる
- Zenn記事 + スライドで他者に説明できる (= Level 2 達成)

## ユーザーの背景

- 26卒、未経験エンジニア
- Go の基本文法を学習済み (vibeコーディング出身、エラーを自力で直せない状態から脱却中)
- 今後インジェスチョン (ETL系) 業務に入る予定
- 「自分で書ける・自分で直せる」が裏ゴール

## 学習計画

`learning/plan.md` を参照。Level 2 (Zenn記事 + スライド) を **4日 (バッファ1日)** で完成させる。

## 読み方の作法 — 一流エンジニアの5要素

ご主人様の "新卒未経験から一流エンジニアの土台を作る" という意志を尊重して、各ループで以下を意識的に実践する:

1. **比較読み (Comparison Read)** — 読む前に「自分ならどう書くか」を書く
2. **逆設計問い (Counter-design)** — 「なぜこう書かなかったか」を読む
3. **破壊テスト (Break Test)** — わざと壊して観察する (1日1回)
4. **教えるつもり読み (Teach-as-you-read)** — 後輩に説明する想定で再構成
5. **デイリーレビュー (Meta-reflection)** — 1日の終わりに 3+3+3 (学び/課題/疑問)

詳細は `learning/plan.md` の「読み方の作法」セクション。

## 教えるときの方針

global CLAUDE.md (`~/.claude/CLAUDE.md`) を継承し、以下を厳守:

- **原始人モード**: 必要最小限の言葉だけで話す。装飾・前置き・繰り返しサマリ禁止
- **段階別出し**: 情報量が多いときは一気に出さず、1段階ずつ。ユーザーの応答を待ってから次へ
- **簡潔に、質問を待つ**: prefill 解説を並べない
- **Insight は1項目以内**: 学習の焦点を散らさない
- **「いつ使うか」(WHEN) を必ず添える**: HOWだけでイメージは湧かない
- **答えに近すぎるヒントは禁止**: コードの雛形提示はNG、問いかけ・notes誘導で導く
- **他言語との比較は使わない**: 日常生活の比喩 (料理・住所・郵便) を使う
- **4ステップ学習法**: 読む → 書く → 実コード読む → まとめる (各ループ ~1.5h)
- **本番(prod)操作は明示確認**: auto modeでも省略しない

## アーキテクチャ概要 (1行で)

```
Slack API (取得) → service (変換) → Slack API (書込) を、GitHub Actions cron で自動実行する Goアプリ
```

詳細は `spec.md`、使い方は `README.md`。

## ファイル構成

```
.
├── cmd/                     # エントリポイント (bot, simulate)
├── internal/                # 業務ロジック (3層: handler / service / repository)
├── .github/workflows/       # CI/CD (cron + workflow_dispatch)
├── learning/                # 学習成果物 ★ ここで学びを集約
│   ├── plan.md              # 4日のループ計画 + 進捗チェックボックス
│   ├── notes/               # 各コードファイル別の30秒地図
│   └── scrap.md             # 時系列の気づき
├── README.md                # 使い方サマリ
├── spec.md                  # 詳細仕様
└── CLAUDE.md                # このファイル
```

## よく使うコマンド

```bash
go run ./cmd/bot recruit          # ローカルで募集投稿
go run ./cmd/bot announce         # ローカルで集計→発表
go run ./cmd/simulate -n 8        # ローカルで N人参加の擬似実行
go test ./...                     # テスト実行
gofmt -w .                        # 整形
```
